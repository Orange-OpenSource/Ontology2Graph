# Common Utilities Module

This folder contains common utility functions and scripts used throughout the GenGraphLLM framework for argument parsing, logging setup, ontology processing, file validation, and graph analysis operations.

## Python Files Overview

### `utils.py`
Core utility module providing foundational services used across all other modules in the framework.

**Main Features:**
- Command-line argument parsing and configuration
- File-based logger setup and management
- Ontology object extraction and processing (classes, properties)
- TTL (Turtle) syntax validation and error handling

**Key Components:**
- `utils_common/utils.py`: Specialized utilities used by others modules
- Argument parsing and configuration management
- Logging and monitoring infrastructure
- File I/O operations and path management
- TTL validation and error handling mechanisms

**Key Functions:**
- `setup_argument_parser(arguments)`: Sets up CLI argument parsers with special handling for mode arguments
- `setup_logger(log_file, logger_name, level)`: Creates file-based loggers with formatted output
- `retreive_onto_object(ontology, object_type)`: Extracts ontology objects from TTL files
- `check_graph_syntax(file_path, bad_syntax_path, logger)`: Validates TTL syntax using external validator

### `extract_nodes_from_files.py`
Standalone script for extracting meaningful nodes from TTL files and exporting them for analysis.

**Purpose:**
Parses Turtle (TTL) RDF files to extract graph nodes while filtering out metadata and datatype properties, then exports clean node lists to CSV format.

**Process:**
1. Parses TTL files using RDFLib to extract RDF triples
2. Filters out metadata predicates (labels, types, descriptions, etc.)
3. Excludes datatype properties defined in the ontology
4. Creates directed graph using NetworkX with filtered relationships
5. Exports resulting nodes to CSV file for further analysis

**Output:** Creates 'nodes.csv' file containing one node name per row

### `find_duplicates_nodes.py`
Comprehensive analysis tool for identifying duplicate nodes across multiple TTL graph files.

**Purpose:**
Analyzes collections of TTL files to identify nodes that appear in multiple graphs, helping understand node overlap and potential merge opportunities in knowledge graph collections.

**Process:**
1. Scans directory for all TTL files
2. Parses each TTL file using RDFLib to extract RDF triples
3. Filters out metadata predicates and datatype properties
4. Extracts nodes from meaningful relationships in each graph
5. Counts node occurrences across all files
6. Identifies and reports duplicate nodes (appearing in multiple files)

**Use Cases:**
- Analyzing synthetic knowledge graphs generated by LLMs
- Identifying common entities across graph collections
- Preparing data for graph merging operations
- Understanding node distribution patterns

**Output:** Detailed report of duplicate nodes with occurrence counts and file locations

### `validate_ttl.py`
Batch TTL syntax validation tool for quality assurance of RDF files.

**Purpose:**
Validates the syntax of Turtle (TTL) RDF files in a directory using an external TTL validator tool, providing comprehensive validation reports for file collections.

**Process:**
1. Scans specified directory for all files
2. Runs TTL validator tool on each file
3. Captures and reports validation results (stdout/stderr)
4. Provides feedback on syntax correctness for each file

**Use Cases:**
- Quality assurance of generated TTL files
- Batch validation of knowledge graph collections
- Identifying syntax errors before graph processing
- Ensuring RDF compliance in automated workflows

**Output:** Console output showing validation status for each processed file

## Dependencies

### Core Dependencies
- **rdflib**: RDF parsing and graph manipulation
- **networkx**: Graph data structure operations
- **pandas**: Data manipulation and CSV export
- **argparse**: Command-line argument parsing
- **logging**: File-based logging functionality
- **pathlib**: Cross-platform file path handling

### External Tools
- **TTL validator**: External command-line tool for Turtle syntax validation
  - Required for `utils.py` and `validate_ttl.py`
  - Must be accessible in system PATH

## Integration with Framework

### Used by generate_ttl_files
- `utils.py`: CLI configuration, logging, ontology extraction, validation
- `validate_ttl.py`: Quality assurance of generated files

### Used by merge_ttl_files  
- `utils.py`: Merge operation parameters, process tracking, output validation
- `find_duplicates_nodes.py`: Pre-merge duplicate analysis

### Used by display_graphs
- `utils.py`: Visualization mode selection, operation logging

### Standalone Analysis Tools
- `extract_nodes_from_files.py`: Independent node extraction for analysis
- `find_duplicates_nodes.py`: Independent duplicate analysis across collections
- `validate_ttl.py`: Independent batch validation tool


## Output Files

### CSV Exports
- **nodes.csv**: Node lists from `extract_nodes_from_files.py`
- Contains clean node names, one per row
- Used for further analysis or processing

### Log Files
- Generated by `utils.py` logging functions
- Formatted with timestamps and severity levels
- Track operations, errors, and validation results

### Validation Reports
- Console output from validation tools
- Detailed error messages for syntax issues
- File movement logs for invalid TTL files

## Error Handling

### File Operations
- Graceful handling of missing files and directories
- Automatic creation of output directories when needed
- Comprehensive error logging for debugging

### RDF Processing
- Robust parsing with error recovery
- Filtering of malformed triples
- Detailed logging of parsing issues

### External Tool Integration
- Subprocess error capture and reporting
- Validation of external tool availability
- Fallback mechanisms for missing dependencies

## Notes

- **TTL Validator Requirement**: External 'ttl' command-line tool must be installed for validation features
- **File Path Compatibility**: All scripts use pathlib.Path for cross-platform compatibility
- **Memory Efficiency**: Large graph collections are processed iteratively to manage memory usage
- **Unicode Support**: Full UTF-8 support for international character sets in RDF data
